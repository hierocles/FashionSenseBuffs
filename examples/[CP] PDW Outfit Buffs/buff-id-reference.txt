PDW internal buff IDs used in content.json (from kath.weathering Data/Buffs):



SomeWeather1     — "Some Weather Huh?" sludge-walk debuff (Heavy Rain, Hail, Blizzard, etc.)

SomeWeather2     — second sludge-walk variant

Firerain         — Acid rain burn debuff

Badbugs          — Locust swarm debuff

ExistentialDread — Tornado watch dread debuff

AngryWeather1    — Hail anger debuff

AngryWeather2    — Heat wave anger debuff

SmokeySinging    — Wildfire smoke debuff

SmoggySinging1   — Smog debuff



Protection buff IDs (applied via BuffIds):

PDWShader, PDWMask, PDWUVGoggles, PDWKittyMuffs, PDWBugNet



When a protection buff is active, PDW normally skips applying the matching debuff.

RemoveBuffIds clears debuffs that were already present before protection kicked in.



Example CP pack (14 base entries, outdoors only)

------------------------------------------------

All entries set "OutdoorsOnly": true because PDW weather effects only apply outdoors.



Seasonal, indoor, and ESWF variant names are listed in "Aliases" on each base entry

(e.g. Heavy Rain includes "Spring Indoor Rain" and "Heavy Rain Indoor").

Add any unmapped outfit name you see in SMAPI debug logs to the Aliases array.



Base entries:

- Acid Rain, Blizzard, Dry Lightning, Hail, Heat Wave, Heavy Rain

- Locust, Meatballs, Mud Rain, Sandstorm, Smog, Tornado, Tropical Storm, Wildfire
